## Project Overview

This is a VitePress-based personal knowledge base (wiki) for Chinese language content. It uses a date-based note organization system with automatic sidebar generation.

## Development Commands

```bash
# Install dependencies
npm install

# Start development server
npm run docs:dev

# Build for production
npm run docs:build

# Preview production build locally
npm run docs:preview
```

## Architecture

### Note Organization System

Notes are organized by date in `docs/notes/YYYY/MM-DD.md`:
- Each year has its own folder (e.g., `docs/notes/2026/`)
- Notes are named with `MM-DD.md` format (month-day)
- The sidebar is automatically generated by `docs/.vitepress/sidebar.mjs` which scans the filesystem at build time

### Sidebar Generation Logic

The `generateSidebar()` function in `sidebar.mjs`:
- Scans `docs/notes/` for year folders (4-digit names)
- Scans each year folder for `.md` files
- Sorts years and files in reverse chronological order (newest first)
- Auto-collapses previous years, keeps current year expanded
- Groups files under year headings (e.g., "2026年")

### VitePress Configuration

Key settings in `docs/.vitepress/config.mjs`:
- `base: '/wiki/'` — Deployed under `/wiki/` subdirectory on GitHub Pages
- `lang: 'zh-CN'` — Chinese language content
- Sidebar is dynamically generated via `generateSidebar()`
- Edit links point to GitHub repository for easy editing

### Deployment

GitHub Actions workflow (`.github/workflows/deploy.yml`):
- Triggers on pushes to `main` branch
- Builds with `npm run docs:build`
- Deploys `docs/.vitepress/dist` to GitHub Pages
- Uses Node.js 20

### Theme Customization

- Custom theme extends default VitePress theme at `docs/.vitepress/theme/index.mjs`
- Custom styles in `docs/.vitepress/theme/style.css`

## File Templates

When creating new daily notes, use the template at `docs/notes/2026/template.md`:
- Frontmatter: `date: YYYY-MM-DD`
- Sections: 今日主题, 笔记内容, 个人思考, 待跟进
