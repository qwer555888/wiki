## 📝 完整流程

### 第一阶段：准备工作

1. 注册并登录 GitHub（https://github.com）
2. 在 GitHub 新建仓库，命名为 `wiki`，设为 Public
3. 在本地 D 盘新建文件夹 `wiki`

1. 打开 PowerShell，进入文件夹：
cd D:\wiki
2. 启动 Claude Code 全自动模式：
claude --dangerously-skip-permissions
3. 发送以下指令给 Claude Code：

网站标题：Quimaz's Wiki

每篇笔记只有日期作为一级标题，下面留几个空白的 ## 占位供我自己填
侧边栏按年份分组显示日期列表
生成 .github/workflows/deploy.yml 自动部署到 GitHub Pages
执行 npm install 安装 vitepress
生成今天 2026-04-04 的笔记模板

4. 等待 Claude Code 自动完成所有文件生成和依赖安装（约3分钟）

### 第三阶段：推送到 GitHub

1. 配置代理（国内必须，否则 push 失败）：

2. 初始化并推送：
git init
git add .
git commit -m "init: setup VitePress wiki"
git branch -M main

git push -u origin main
3. Push 时会弹出 GitHub 登录窗口，点击 **Sign in with your browser** 授权
4. 授权完成后浏览器会出现空白页，这是正常的，回到终端查看结果

### 第四阶段：开启 GitHub Pages

1. 访问：https://github.com/qwer555888/wiki/settings/pages
2. Source 选择 **GitHub Actions**（有勾即为已选中）

## ⚠️ 容易出错的地方

### 1. 代理问题（最常见）
- **现象**：git push 超时或失败
- **原因**：国内访问 GitHub 被墙，Git 默认不走系统代理
- **解决**：手动配置 Git 代理端口
git config --global http.proxy http://127.0.0.1:你的端口号
- **查看端口**：打开代理软件 → 设置 → Local port

### 2. GitHub Pages 部署失败（404错误）
- **现象**：Actions 里显示红色 ❌
- **原因**：GitHub Pages 的 Source 没有正确设置
- **解决**：去 Settings → Pages → Source 选 GitHub Actions，然后重新 push

### 3. 授权后浏览器空白页
- **现象**：git push 弹出登录窗口，授权后浏览器一片空白
- **原因**：这是正常现象，授权回调页没有内容
- **处理**：直接回到终端，看 push 是否成功

### 4. 代理模式不需要切换
- Smart 模式 + 手动配置 Git 代理端口 = 完美解决
- 不需要切换到 Global 模式
